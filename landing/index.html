How it works

From error to replay in seconds

🔴
1 — Record

SDK keeps a circular buffer of the last 30–60s. Always recording, never sending noise.

→
⚡
2 — Capture

Error detected. Buffer flushes to the backend automatically — or the user hits "Report a bug".

→
▶
3 — Replay

Open the dashboard, hit play. See exactly what the user did — DOM, network, console, all in sync.

Everything you need for session replay

Without the cloud bill.

🎬

DOM Recording

rrweb captures every click, scroll, input, and navigation. Pixel-perfect replay without screenshots.

⚡

Error-triggered upload

Circular buffer records the last 30–60s. Sends automatically only when an error is detected. No noise.

🐛

User reports

Floating widget in Shadow DOM. User types a title, hits send. Replay + report land together in your dashboard.

🌐

Network capture

Every request logged, including canceled and aborted. Synced with the replay timeline.

🖥️

Console capture

log, warn, error captured via rrweb console plugin. Click an error to jump to that moment in the replay.

🚀

One-command deploy

SQLite + local filesystem. No Redis, no PostgreSQL, no Kafka. Runs comfortably on a $6 VPS.
